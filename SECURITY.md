# Security Policy

## Supported versions

The latest `main` branch and the most recent published template package are the supported targets for security fixes.

## Reporting a vulnerability

Please do not open public issues for suspected security vulnerabilities.

Instead:

1. Email the maintainer or use the private reporting channel configured for the repository.
2. Include reproduction steps, affected files, and any proof-of-concept details.
3. Allow time for triage before public disclosure.

If the issue affects generated projects, include the template version and the generated project name you used.
